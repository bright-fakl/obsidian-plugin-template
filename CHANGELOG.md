## How to Update This Changelog

When releasing a new version:

1. Move items from `[Unreleased]` to a new version section
2. Update the version number and date
3. Add a comparison link at the bottom
4. Follow these categories:
   - **Added** - New features
   - **Changed** - Changes in existing functionality
   - **Deprecated** - Soon-to-be removed features
   - **Removed** - Removed features
   - **Fixed** - Bug fixes
   - **Security** - Security fixes

### Example Entry

```markdown
## [1.0.0] - 2024-01-15

### Added
- New sidebar view for browsing notes
- Support for custom keyboard shortcuts

### Changed
- Improved performance of text transformation
- Updated settings UI layout

### Fixed
- Fixed issue with unicode characters in transformations
- Resolved settings not saving on mobile
```
